# Phase 1 验收清单

## 功能验收（20 项）

### 记账功能
- [ ] 1. 中文记账：用户发送"今天午餐30元"，系统正确记录支出
- [ ] 2. 收入记录：用户发送"收到工资8000"，系统识别为收入
- [ ] 3. 混合意图：用户发送"午饭30，帮我看这个月餐饮总额"，先回答查询再记账
- [ ] 4. 去重确认：同一次会话中重复记账应触发确认提示
- [ ] 5. 数据修正：用户发送"午餐其实是35不是30"，系统更新记录

### 查询功能
- [ ] 6. 时间范围查询：今天/昨天/这周/上周/这个月/上个月
- [ ] 7. 分类汇总：按餐饮/交通等分类统计金额
- [ ] 8. 预算使用率：查询某分类预算剩余百分比

### 分类与意图识别
- [ ] 9. 收入/支出方向识别准确率 >= 98%
- [ ] 10. 一级分类命中率 >= 95%
- [ ] 11. 低置信度分类进入 needs_review，不静默误分类
- [ ] 12. 用户纠正后别名更新，后续同类表达命中率提升

### 记忆系统
- [ ] 13. 短期记忆压缩：对话超过 30 轮时自动摘要
- [ ] 14. mem0 不可用时系统降级运行，不崩溃

### 安全与可靠性
- [ ] 15. 无 API Key 请求返回 401
- [ ] 16. 错误 API Key 请求返回 401
- [ ] 17. 健康检查端点 `/health` 返回 ok

### 可观测性
- [ ] 18. 日志包含 trace_id
- [ ] 19. 日志中金额已脱敏（显示为 XXX.XX）

### 数据库
- [ ] 20. SQLite WAL 模式启用
- [ ] 21. 分类种子数据已加载（16 个一级分类）
